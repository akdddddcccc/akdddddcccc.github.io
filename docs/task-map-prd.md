# Task Map 原型 PRD

## 1. 项目定位

Task Map 是一个面向个人创作、设计项目、学习计划和复杂任务拆解的结构化任务规划工具。它不是全自动管家，而是把「任务拆解」「层级结构」「时间范围」放到同一个可视化工作台里，让用户先看清任务关系，再决定如何执行。

当前版本作为作品集网站中 `vibe coding` 分类的交互原型展示，目标是让访问者可以直接体验核心交互，而不依赖登录和云端储存。

## 2. 目标用户

- 需要把模糊目标拆成可执行步骤的学生、设计师、自由职业者和个人开发者。
- 经常在脑图、文档、日历、待办工具之间来回切换的人。
- 想快速评估一个项目结构和时间压力的人。

## 3. 核心问题

传统待办工具太像表格，适合记录结果，不适合从零开始思考结构；传统思维导图适合拆解，但弱化了时间和执行状态；日历适合精确日期，但不适合表达一组任务之间的结构关系。

Task Map 的核心机会是：先用思维导图表达结构，再用甘特时间轴表达时间范围，同时让 AI 只提供拆解建议和初步时间分配，最终控制权仍在用户手里。

## 4. 产品原则

- AI 提供思路，不代替用户管理人生。
- 精确到日和小时的时间只用于 DDL，任务周期主要通过时间轴拖动确定。
- 结构编辑优先，交互要接近幕布式大纲和脑图的效率。
- 思维导图与时间轴上下排列，避免两个横向大画布互相挤压。
- Demo 版本优先证明交互和概念，不做登录、账户、云同步。

## 5. 核心功能范围

### 5.1 AI 拆解建议

用户输入一个模糊目标后，AI 根据当前选中的任务节点生成若干同级或下级子任务建议。

AI 输出内容包括：

- 子任务标题。
- 简短说明。
- 同级子任务的粗略时间划分建议。
- 默认 DDL 建议。

AI 不负责：

- 自动安排用户全天日程。
- 自动决定任务优先级。
- 自动修改已锁定的用户时间范围。
- 像管家一样持续提醒、催办或替用户执行任务。

### 5.2 思维导图

思维导图是任务结构的主编辑区。

基础交互：

- 点击节点：选中任务，并同步高亮时间轴中的对应任务。
- 悬停节点：显示下一级子任务预览。
- 新增子节点：在当前节点下创建新任务。
- 新增同级节点：在当前节点后创建同级任务。
- 删除节点：删除当前节点及其子节点。
- 拖拽节点到另一个节点：改变父子层级关系。

键盘交互：

- `Enter`：新增同级任务。
- `Tab`：将当前任务降级为上一个任务的子任务。
- `Shift + Tab`：将当前任务升级到父级同层。
- `Delete` / `Backspace`：删除当前任务；当用户正在输入标题时不触发删除。

### 5.3 时间轴 / 甘特结构

时间轴用于表达任务的时间范围，不作为精确日历使用。

基础交互：

- 每个任务以甘特条显示开始和结束范围。
- 父子层级通过缩进体现。
- 点击时间轴任务：同步选中思维导图节点。
- 悬停时间轴任务：同步显示结构相关节点。
- 拖动甘特条：修改任务开始和结束范围。
- 子任务的时间范围用于帮助用户判断父级任务是否合理。

时间规则：

- DDL 使用精确日期和小时。
- 普通任务周期主要通过拖动时间轴确定。
- 已锁定任务不应被 AI 自动覆盖。
- 若子任务超出父任务范围，系统显示冲突提示。

### 5.4 导出

Demo 需要支持导出：

- 可交互 HTML，用于离线展示或发送给他人体验。
- PDF，用于作品集展示、归档和静态汇报。

导出内容应包含任务标题、层级、时间范围、状态和说明。

## 6. 数据结构草案

每个任务节点至少包含：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | string | 任务唯一标识 |
| `parentId` | string / null | 父任务 ID |
| `title` | string | 任务标题 |
| `note` | string | 任务说明 |
| `start` | string | 时间范围开始日期 |
| `end` | string | 时间范围结束日期 |
| `ddl` | string | 精确到日和小时的截止时间 |
| `mode` | `auto` / `locked` | 时间是否允许 AI 建议覆盖 |
| `done` | boolean | 是否完成 |

## 7. 当前 Demo 验收标准

- 页面中只保留「思维导图」与「时间轴 / 甘特结构」两个核心区域。
- 思维导图支持节点新增、删除、层级调整、拖拽变更父子关系。
- 思维导图支持 `Enter`、`Tab`、`Shift + Tab`、`Delete` / `Backspace` 快捷操作。
- 时间轴以甘特图形式展示层级任务。
- 时间轴任务可拖动修改时间范围。
- DDL 使用精确到日和小时的输入。
- AI 面板的文案明确表达「提供拆解建议」而非「全自动管理」。
- 支持导出交互 HTML 和 PDF。
- 作品集移动端不展示 `vibe coding` 分类，因此此 Demo 暂不要求移动端适配。

## 8. 后续版本方向

- 接入真实 AI API，根据用户输入和当前节点生成更稳定的拆解方案。
- 增加本地储存或云端储存。
- 增加历史版本和撤销 / 重做。
- 增加更精细的拖拽手柄，用于分别调整任务开始和结束时间。
- 支持从 Markdown、大纲、CSV 导入任务。
- 支持把导出的交互 HTML 作为独立作品或项目附件发布。
- 增加冲突分析，例如任务重叠过多、父子范围不一致、DDL 过密。

## 9. 非目标

- 不做完整日历应用。
- 不做团队协作系统。
- 不做账号体系和权限系统。
- 不做自动执行任务的 AI Agent。
- 不做移动端优先体验。
